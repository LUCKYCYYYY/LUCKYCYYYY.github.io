# 指令字长

  • 指令字长越长，则占用主存空间就越大，所耗费的访存时间也就越长
  • 根据指令字长与机器字长的关系，指令可分为
    • 半字长指令：CPUCPUCPU 进行一次访问，可读取 222 条
    • 单字长指令：CPUCPUCPU 进行一次访问，可读取 111 条
    • 多字长指令：需要进行多次访问

# 堆栈寻址

存储器堆栈

内存基于地址访问，所以需要设置一个堆栈指针寄存器 SPSPSP

  • 入栈 SP=SP−1,M[SP]=RSP=SP-1,M[SP]=RSP=SP−1,M[SP]=R
  • 出栈 R=M[SP],SP=SP+1R=M[SP],SP=SP+1R=M[SP],SP=SP+1

寄存器堆栈

对比

  • 寄存器堆栈栈顶固定不动，存储寄存器堆栈栈顶随着对战操作而移动
  • 进行堆栈操作时，寄存器堆栈中的数据移动，存储器堆栈中的数据不动
  • 寄存器堆栈速度快，但容量有限；存储器速度较慢，但是容量大
  • 寄存器堆栈必须采用堆栈指令进行控制，存储器堆栈则不一定

# MIPS 指令系统

MIPSMIPSMIPS 体系结构是 RISCRISCRISC 体系构架

MIPS 指令集有以下特点：

  • 简单的 Load/Store 结构

  • 易于流水线 CPU 设计

  • 易于编译器开发

  • MIPS 指令的寻址方式非常简单，每条指令的操作也非常简单

# MIPS 指令格式

RRR 型指令：

  • 操作数只能来自寄存器，运算结果也使能存入寄存器
    • 若是双目运算，rsrsrs 和 rtrtrt 字段分别是第一和第二源操作数，rdrdrd 字段为目的操作数
    • 若是移位运算，则表示对 rtrtrt 的内容进行移位，移位位数由 shamtshamtshamt 字段决定（shamtshamtshamt 是移位变量，其他指令无效）
  • OPOPOP 字段为 000000000000000000，具体操作由 functfunctfunct 字段决定

III 型指令：

  • 立即数型指令
  • 若是双目运算，则将寄存器 rsrsrs 和立即数字段分别作为第一和第二操作数，，将结果送入寄存器 rtrtrt
  • 若是 LOAD/STORELOAD/STORELOAD/STORE 指令，则将寄存器 rsrsrs 值和立即数字段的值相加生成存储单元地址
  • 若是条件分支指令，则对 rsrsrs 和 rtrtrt 寄存器中的数据进行规定的判别运算，并根据结构决定是否转移
    • 当转移发生时，目标地址采用相对寻址方式获得，即将 PC+4PC+4PC+4 的值和立即数相加得到

JJJ 型指令：

  • 无条件转移指令
  • 目标地址由 PC+4PC+4PC+4 的高 444 位与 262626 位直接地址 addressaddressaddress 经左移 222 位后的值拼接得到

# MIPS 指令寻址方式

MIPSMIPSMIPS 指令格式的寻址方式只有 555 种

  • RRR 型指令的寻址方式只有寄存器寻址
  • III 型指令的寻址方式有寄存器寻址、立即数寻址、基址寻址（偏移寻址）、相对寻址
  • JJJ 型指令的寻址方式只有伪直接寻址
