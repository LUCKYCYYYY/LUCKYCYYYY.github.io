# 溢出及检测

# 溢出的概念

因为计算机的字长是有限的，因此所能表达的数据范围也是有限的

当运算结果超出所能表示的数据范围时，就会出现溢出；溢出会导致运算结果错误

# 溢出检测

方法一：根据操作数和运算结果的符号位是否一致进行检测

证明

假设机器字长为 n+1n+1n+1 位

因此补码所能表示的范围是

−2n≤真值<2n-2^n\leq \text{真值}< 2^n −2n≤真值<2n

当真值 xxx 和 yyy 同为负数时：

−2n≤x<0−2n≤y<0}⇒0>x+y≥−2n+1\left.\begin{matrix} -2^n\leq x<0 \\ -2^n\leq y<0 \end{matrix}\right\}\Rightarrow 0>x+y\geq -2^{n+1} −2n≤x<0−2n≤y<0​}⇒0>x+y≥−2n+1

可能负溢

当真值 xxx 和 yyy 同为正数时：

0<x<2n0<y<2n}⇒0<x+y<2n+1\left.\begin{matrix} 0< x<2^n \\ 0< y<2^n \end{matrix}\right\}\Rightarrow 0<x+y<2^{n+1} 0<x<2n0<y<2n​}⇒0<x+y<2n+1

可能正溢

方法二：

根据运算过程中最高数值位的进位与符号位的进位是否一致进行判断。不同可判定为产生溢出，相同可判定为没有产生溢出

方法三：

利用变形补码（具有 2 位符号位的补码）的符号位进行判断

  • 双符号位为 00 时，表示正数
  • 双符号位为 11 时，表示负数
  • 双符号位为 01 时，表示正溢出
  • 双符号位为 10 时，表示负溢出

32 位全先行进位 ALU

  • 设计思路：

    • 每 444 位 用 111 片 741817418174181 处理
    • 每 444 片 741817418174181（16 位）用 111 片 741827418274182 实现组间先行进位
    • 对于 323232 位，需要 两级 741827418274182 实现全先行进位（即组内和组间均并行进位）

  • 计算：

    • 74181 数量：

      32 位4 位/片=8 片\frac{32 位}{4 位/片}=8 片 4 位/片32 位​=8 片

    • 74182 数量：

      • 第一级：每 444 片 741817418174181 配 111 片 741827418274182 → 222 片（覆盖 888 片 741817418174181）
      • 第二级：用 111 片 741827418274182 汇总前两级 741827418274182 的进位 → 111 片

# 加减法运算

设有两个浮点数 X=Mx×2ExX=M_x\times 2^{E_x}X=Mx​×2Ex​ 和 Y=My×2EyY=M_y\times 2^{E_y}Y=My​×2Ey​，且 Ex>EyE_x>E_yEx​>Ey​

若 X±YX\pm YX±Y 的结果是 SSS

S=X±Y=Ms×2EsS=X\pm Y=M_s\times 2^{E_s} S=X±Y=Ms​×2Es​

其中

Es=Ex,Ms=Mx±(My>>(Ex−Ey))E_s=E_x,M_s=M_x\pm(My>>(E_x-E_y)) Es​=Ex​,Ms​=Mx​±(My>>(Ex​−Ey​))

浮点数加减法运算的步骤：

  1. 零操作数检查：一个操作数为 000，则不必运算，节省运算时间

  2. 两操作数对阶：让小的阶码向大的阶码看齐

    • 求阶差：通过对两个阶码进行减法运算实现，求出两个阶码的具体差值

    • 若 Ex>EyE_x>E_yEx​>Ey​，则将浮点数 YYY 的尾数右移 Ex−EyE_x-E_yEx​−Ey​

      若 Ex<EyE_x<E_yEx​<Ey​，则将浮点数 XXX 的尾数右移 Ey−ExE_y-E_xEy​−Ex​

      通常在尾数右移时通常将最低位的移除位暂时保留，称为保留附加位

      在 IEEE754IEEE754IEEE754 种通常包括 333 个保留附加位，从左到右分别为保留位、舍入位、粘位（舍入位右侧数据有非 0 数据则粘位为 111，否则为 000

  3. 尾数运算：以双符号位的补码形式进行加减法运算

  4. 结果的规格化：当尾数不是 11.0.... 或 00.1... 的形式时，应进行相应的规格化处理

    • 当尾数符号位为 010101 或 101010 时，运算结果上溢，需要向右规格化，且只需将尾数算数右移一位，同时将结果的阶码值加 111
    • 当尾数符号位为 11.1... 或 00.0... 时需要向左规格化，而且左移次数不固定

  5. 舍入：在尾数进行向右规格化操作时，尾数的末尾的几位会因超出计算机字长而被丢掉

    • 末位恒置 111 法：只要因移位而丢失的位种有一位是 111，就把运算结果的最低为置 111，而不管最低位原来是 000 还是 111
    • 000 舍 111 入法：当丢失位数的最高位是 111 时将尾数的末位加 111，类似于十进制数的四舍五入

    如果舍入操作破坏了规格化，则需重新进行规格化

  6. 溢出判断：当阶码符号位为 010101 和 101010 时才表示运算结果溢出

    • 在左规的时候，判断阶码是否下溢，若发生下溢（符号位为 101010），可认为结果为 000
    • 在右规的时候，判断阶码是否上溢，若发生上溢（符号位为 010101），可认为结果为 ∞\infty∞
